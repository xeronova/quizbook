js/questions.js 파일의 모든 퀴즈 문제에 대해 **정답의 사실적 정확성**을 검증하라.

verify-quiz 명령이 가이드라인 형식을 점검하는 것과 달리, 이 명령은 각 문제의 정답이 **실제로 맞는지**를 확인한다.

## 검증 대상

js/questions.js의 QUESTIONS 배열에 있는 모든 문제.

## 검증 방법

각 문제에 대해 아래 4단계를 수행한다:

### 1단계: 정답 확인
- `correctAnswer` 인덱스가 가리키는 `options` 항목이 질문에 대한 올바른 답인지 확인한다.
- 인덱스가 범위를 벗어나면 (0~3 외) 즉시 오류로 보고한다.

### 2단계: 오답 선택지 검증
- 나머지 3개 선택지가 실제로 오답인지 확인한다.
- 오답 중 정답이 될 수 있는 항목이 있으면 보고한다.

### 3단계: 해설 일치 확인
- `explanation`의 내용이 정답과 일치하는지 확인한다.
- 해설에 사실과 다른 내용이 포함되어 있으면 보고한다.

### 4단계: 웹 교차 검증
- 카테고리별로 에이전트를 병렬 실행하여 검증한다.
- 가능하면 웹 검색을 통해 사실 확인을 수행한다.
- 웹 검색이 불가능할 경우 학습 데이터 기반으로 검증하되, 확신도를 표시한다.

## 검증 결과 분류

각 문제를 다음 중 하나로 분류한다:

- **PASS**: 정답이 정확하고, 해설이 일치하며, 오답도 명확히 틀림
- **FAIL**: 정답이 틀렸거나, correctAnswer 인덱스가 잘못됨
- **WARN**: 정답은 맞지만 해설에 부정확한 내용이 있거나, 오답 중 논란의 여지가 있는 항목이 있음
- **UNCERTAIN**: 확실하게 판단할 수 없음 (확신도 표시)

## 출력 형식

### 카테고리별 검증 결과

카테고리별로 묶어서 보고한다:

```
## 한국사 (N/M 통과)

| ID | 질문 요약 | 정답 | 결과 | 비고 |
|----|----------|------|------|------|
| 1  | 조선 건국한 왕 | 이성계 | PASS | - |
| 4  | 거북선을 만든 장군 | 이순신 | WARN | 해설 부정확: ... |
```

### 문제 상세 (FAIL/WARN만)

FAIL 또는 WARN 판정을 받은 문제에 대해 상세 보고:

```
### [FAIL] ID 6: 삼국 중 가장 넓은 영토
- 현재 정답: 고구려 (index 0)
- 문제점: <구체적 설명>
- 수정 제안: <correctAnswer 또는 question 수정안>
```

### 전체 요약

```
전체: N문제
  PASS: A개
  WARN: B개
  FAIL: C개
  UNCERTAIN: D개
정확도: A/N × 100%
```

## 작업 방식

- 카테고리별로 병렬 에이전트를 실행하여 검증 속도를 높인다.
- 4개 카테고리: 한국사, 과학, 지리, 예술과 문화
- 각 에이전트는 해당 카테고리의 모든 문제를 검증하고 결과를 반환한다.

## 주의사항

- 이 명령은 읽기 전용이다. 파일을 수정하지 마라.
- FAIL이 발견되면 수정 제안을 포함하되, 사용자 확인 전까지 적용하지 마라.
- 사용자가 수정을 요청하면 그때 js/questions.js를 수정한다.
