개별 학생의 퀴즈 성적 보고서를 생성하라.

## 입력 파라미터

- **$ARGUMENTS**: 학생 이름
  - 생략 시: `data/roster.json`의 학생 목록을 보여주고 선택하게 한다
  - 존재하지 않는 이름이면: "❌ '<이름>'은(는) 등록되지 않은 학생입니다." 출력 후 중단

## 사전 조건

- `data/students/<이름>.json` 파일이 존재해야 한다. 없으면 `/teacher-import`를 먼저 실행하라고 안내한다.
- `data/roster.json`이 존재해야 한다. 없으면 `/teacher-setup`을 먼저 실행하라고 안내한다.

## 실행 절차

---

### 1단계: 학생 데이터 로드

`data/students/<이름>.json`을 읽고 데이터를 분석한다.

**로드 확인:**
```
📂 학생 데이터 로드: <이름>
  - 파일: data/students/<이름>.json
  - 기록 수: N개
  - 마지막 가져오기: <날짜>
```

**성공 조건**: 파일이 존재하고 유효한 JSON
**실패 시**: "❌ 1단계 실패: 학생 데이터 로드 실패 — <구체적 원인>" 출력 후 중단

---

### 2단계: 종합 성적 분석

results 배열을 분석하여 다음 지표를 계산한다:

| 지표 | 계산 방식 |
|------|----------|
| 총 게임 수 | results.length |
| 총 정답 수 | Σ(correct) |
| 총 문제 수 | Σ(total) |
| 평균 점수 | Σ(score) / 게임 수 |
| 최고 점수 | max(score) |
| 최저 점수 | min(score) |
| 평균 정답률 | (총 정답 / 총 문제) × 100 |
| 최고 연속 정답 | max(streak) |
| 가장 많이 플레이한 모드 | mode별 빈도 |

---

### 3단계: 카테고리별 분석

results에서 카테고리별 성적을 추출한다 (카테고리별 모드 결과가 있는 경우):

```
📊 카테고리별 성적

| 카테고리      | 게임 수 | 평균 정답률 | 평균 점수 | 판정   |
|--------------|--------|-----------|---------|--------|
| 한국사        |   ?    |    ?%     |   ?점   | 강점 💪 |
| 과학          |   ?    |    ?%     |   ?점   | 보통 📊 |
| 지리          |   ?    |    ?%     |   ?점   | 약점 📚 |
| 예술과 문화    |   ?    |    ?%     |   ?점   | 강점 💪 |
```

**판정 기준:**
- 💪 강점: 평균 정답률 80% 이상
- 📊 보통: 평균 정답률 50~79%
- 📚 약점: 평균 정답률 50% 미만
- ➖ 데이터 없음: 해당 카테고리 기록 없음

---

### 4단계: 성장 추이 분석

results를 시간순으로 정렬하여 성장 추이를 분석한다:

```
📈 성장 추이 (최근 10게임)

| 순서 | 날짜       | 모드     | 점수 | 정답률 | 변화 |
|------|-----------|---------|------|-------|------|
| 1    | 01/10     | 전체 도전 | 120  | 75%   | —    |
| 2    | 01/12     | 스피드   | 85   | 60%   | ↓    |
| 3    | 01/15     | 카테고리 | 95   | 80%   | ↑    |
| ...  |           |         |      |       |      |
```

**추이 요약:**
- 점수 추세: 상승 📈 / 하락 📉 / 정체 ➡️ (최근 5게임 vs 이전 5게임 비교)
- 정답률 추세: 상승 / 하락 / 정체

---

### 5단계: 종합 평가 및 조언

분석 결과를 바탕으로 종합 평가를 작성한다:

```
📝 종합 평가

🏆 등급: <S/A/B/C/D> (평균 정답률 기준)
  S: 90% 이상  A: 80~89%  B: 70~79%  C: 60~69%  D: 60% 미만

💪 강점:
  - <카테고리>에서 우수한 성적 (정답률 ?%)
  - <기타 강점>

📚 개선 필요:
  - <카테고리>에서 보충 필요 (정답률 ?%)
  - <기타 개선점>

💡 추천:
  - <구체적 학습 추천 사항>
```

---

### 6단계: 보고서 파일 생성

분석 결과를 마크다운 파일로 저장한다:

- **파일 경로**: `data/reports/<이름>_report.md`
- **파일 내용**: 2~5단계 분석 결과를 마크다운 형식으로 정리
- **파일 헤더**:

```markdown
# 📊 학생 성적 보고서: <이름>

생성일: <YYYY-MM-DD HH:MM:SS>
데이터 기간: <최초 게임 날짜> ~ <최근 게임 날짜>

---
```

**성공 조건**: 보고서 파일이 생성됨
**실패 시**: 화면에 결과는 표시하되 "⚠️ 파일 저장 실패" 경고 출력

---

### 7단계: 결과 요약

```
✅ 학생 보고서 생성 완료

👤 학생: <이름>
🏆 등급: <등급>
📊 평균 정답률: ?%
📈 추세: <상승/하락/정체>
📁 보고서: data/reports/<이름>_report.md
```

---

## 오류 처리 정책

- 학생 데이터가 없으면 등록된 학생 목록을 표시하고 중단
- results가 빈 배열이면 "데이터가 부족합니다" 안내 후 중단
- 카테고리 분석이 불가능한 경우 (모드 정보 없음) 해당 섹션을 "데이터 없음"으로 표시

## 주의사항

- 읽기 전용 작업이다. 학생 데이터 파일을 수정하지 마라.
- 보고서 파일(`data/reports/`)만 생성/덮어쓴다.
- 기존 보고서가 있으면 사용자에게 덮어쓸지 물어본다.
