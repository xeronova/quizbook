js/questions.js 파일에 새 퀴즈 문제를 추가하라.

## 입력 파라미터

- **카테고리**: $ARGUMENTS (첫 번째 인자)
  - 허용값: 한국사, 과학, 지리, "예술과 문화"
  - 생략 시 사용자에게 물어볼 것
- **난이도**: $ARGUMENTS (두 번째 인자)
  - 허용값: easy, medium, hard
  - 생략 시 사용자에게 물어볼 것

## 작업 절차

### 1단계: 기존 문제 분석
- js/questions.js를 읽어 현재 총 문제 수와 마지막 id를 확인한다.
- 해당 카테고리 + 난이도 조합의 기존 문제를 파악하여 중복을 피한다.

### 2단계: 문제 생성
- 새 문제 1개를 생성한다.
- 반드시 아래 형식을 따를 것:

```js
{
  id: <마지막 id + 1>,
  category: "<카테고리>",
  difficulty: "<난이도>",
  question: "<질문>",
  options: ["<선택지1>", "<선택지2>", "<선택지3>", "<선택지4>"],
  correctAnswer: <0~3>,
  explanation: "<해설>"
}
```

### 3단계: 검증 가이드라인 적용 (반드시 준수)
생성한 문제를 CLAUDE.md의 "퀴즈 문제 교차 검증 가이드라인"에 따라 자체 점검한다.

**체크리스트:**
1. **정답 유일성**: 정답이 하나뿐인가? 다른 해석 가능 시 조건을 명시했는가? (예: "면적 기준", "2024년 기준")
2. **최상급 표현 기준**: '가장 큰', '최초의' 등 최상급 표현에 측정 기준이 명시되어 있는가?
3. **시간과 범위**: 변할 수 있는 정보에 시점이 명시되어 있는가? 지리적·분류적 범위가 한정되어 있는가?
4. **교차 검증**: 정답이 학술적으로 확립된 사실인가? 논란이 있는 내용은 주류 학설 기준인가?

위반 항목이 있으면 문제를 수정하여 모든 기준을 통과시킨 후 진행한다.

### 4단계: 사용자 확인
- 생성한 문제를 사용자에게 보여주고 승인을 받는다.
- 검증 결과(4개 항목 통과 여부)도 함께 표시한다.

### 5단계: 파일에 삽입
- 승인을 받은 후에만 js/questions.js 파일을 수정한다.
- 해당 카테고리 섹션의 마지막 문제 뒤에 삽입한다.
- 기존 코드 스타일(들여쓰기 2칸, 쉼표 위치 등)을 그대로 따른다.

## 출력 형식

```
📝 새 문제 생성

카테고리: <카테고리>
난이도: <난이도>
ID: <번호>

Q. <질문>
  A. <선택지1>
  B. <선택지2>
  C. <선택지3>
  D. <선택지4>
정답: <정답 레이블>
해설: <해설>

✅ 검증 결과
1. 정답 유일성: 통과/위반
2. 최상급 표현: 통과/해당없음
3. 시간과 범위: 통과/해당없음
4. 교차 검증: 통과/위반
```

## 주의사항
- 사용자 승인 없이 파일을 수정하지 마라.
- 오답 선택지도 그럴듯하게 만들되, 정답과 혼동되지 않도록 한다.
- explanation은 정답의 근거를 간결하게 1~2문장으로 작성한다.
- 카테고리가 허용값에 없으면 에러 메시지를 출력하고 중단한다.
