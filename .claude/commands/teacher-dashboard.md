학급 전체 현황을 시각적으로 한눈에 파악할 수 있는 HTML 대시보드를 생성하라.

## 입력 파라미터

- **$ARGUMENTS**: 없음 (전체 학급 대상)

## 사전 조건

- `data/roster.json`이 존재해야 한다. 없으면 `/teacher-setup`을 먼저 실행하라고 안내한다.
- `data/students/` 디렉토리에 최소 1명 이상의 학생 데이터가 있어야 한다.

## 실행 절차

---

### 1단계: 전체 학생 데이터 로드

`data/roster.json`의 학생 목록을 읽고, 각 학생의 JSON 데이터를 로드한다:

```
📂 학급 데이터 로드

등록 학생: N명
데이터 로드 성공: M명
데이터 없음: K명 (<이름 목록>)
```

데이터가 없는 학생은 경고 후 분석에서 제외한다.

**성공 조건**: 1명 이상의 학생 데이터 로드 완료
**실패 시**: "❌ 1단계 실패: 분석할 학생 데이터가 없습니다" 출력 후 중단

---

### 2단계: 학생별 메트릭 계산

각 학생에 대해 다음을 계산한다:
- 총 게임 수, 평균 점수, 평균 정답률, 최고 연속 정답
- 추세 (전반기 vs 후반기 정답률 차이)
- 카테고리별 정답률
- 등급 (S/A/B/C/D)

---

### 3단계: dashboard.html 생성

`data/reports/dashboard.html`에 자체 완결형 HTML 대시보드를 생성한다.

**대시보드 구성 (6개 섹션):**

1. **헤더**: 제목, 생성일, PDF 다운로드 버튼(`window.print()`), 다크모드 토글
2. **학급 개요**: 4개 메트릭 카드 (총 학생 수, 학급 평균 정답률, 총 게임 수, 평균 게임 수/학생)
3. **학생별 성적 테이블**: 순위/이름/게임수/평균점수/정답률/최고연속/추세/등급 — 컬럼 클릭 시 정렬
4. **카테고리별 분석**: 4개 카테고리 카드, 학생별 수평 막대 차트, 학급평균선 표시
5. **정답률 분포**: 5구간 히스토그램 (S/A/B/C/D), 학생 이름 표시
6. **액션 아이템**: 즉시 조치 / 모니터링 / 긍정적 사항

**디자인 요구사항:**
- 퀴즈북과 동일한 색상 체계 (--primary: #4f46e5 등)
- 다크모드 지원 (`[data-theme="dark"]`)
- max-width: 1100px, 카드 기반 레이아웃
- `@media print`: 버튼 숨김, 배경 흰색, break-inside: avoid, @page A4

**데이터 임베딩:**
- 학생 데이터를 JavaScript 상수로 HTML 파일에 직접 임베드한다
- 외부 파일 참조 없이 단독 실행 가능해야 한다

---

### 4단계: 브라우저에서 열기

생성된 HTML 파일을 브라우저에서 연다:

```bash
open data/reports/dashboard.html
```

---

### 5단계: 최종 보고

```
✅ 학급 대시보드 생성 완료

👥 분석 학생: N명
📊 학급 평균 정답률: ?%
📁 대시보드: data/reports/dashboard.html

💡 사용법:
- 브라우저에서 직접 열어 확인
- PDF 다운로드 버튼으로 인쇄/저장
- 다크모드 토글로 테마 전환
- 테이블 헤더 클릭으로 정렬
```

---

## 오류 처리 정책

- 학생 데이터가 0명이면 즉시 중단
- 카테고리 데이터가 부족한 학생은 해당 카테고리를 제외하고 계속 진행
- 기존 dashboard.html이 있으면 덮어쓴다

## 주의사항

- 학생 데이터 파일은 읽기 전용으로 수정하지 마라
- `data/reports/dashboard.html`만 생성/덮어쓴다
- HTML 파일은 외부 의존성 없이 자체 완결형이어야 한다
